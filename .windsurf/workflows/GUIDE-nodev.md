# คู่มือใช้งาน Windsurf Workflow สำหรับผู้ใช้ทั่วไป

> ไม่ต้องรู้โค้ด ไม่ต้องรู้เทคนิค
> แค่บอก Windsurf ว่าเกิดอะไรขึ้น หรืออยากให้ทำอะไร

---

## วิธีใช้คู่มือนี้

1. เปิด Windsurf → กด `Cmd+L` (Mac) หรือ `Ctrl+L` (Windows)
2. หาสถานการณ์ที่ตรงกับปัญหาของคุณในคู่มือ
3. คัดลอกข้อความตัวอย่าง → วางในช่องพิมพ์ → กด Enter
4. Windsurf จะทำงานให้

## 1. กดปุ่มแล้วไม่มีอะไรเกิดขึ้น

### สถานการณ์
กดปุ่ม "บันทึก" / "ส่ง" / "อนุมัติ" แล้วไม่มี action อะไรเลย — ไม่ขึ้น success, ไม่ขึ้น error, ไม่มีอะไรเปลี่ยนแปลง

### วิธีบอก Windsurf
```
/debug เมื่อกดปุ่มบันทึกที่หน้า [ชื่อหน้า] แล้วไม่มี action อะไรเกิดขึ้นเลย
ไม่ขึ้นข้อความสำเร็จ ไม่ขึ้น error ไม่มีอะไรเปลี่ยนแปลง
ข้อมูลที่กรอกคือ [ระบุข้อมูลที่กรอกถ้าได้]
```

### ตัวอย่างจริง

**กรณี: กดบันทึกหน้า Setting > Asset Group แล้วไม่มีอะไรเกิดขึ้น**
```
/debug เมื่อเข้าเมนู Setting > Asset Group แล้วกดปุ่มบันทึก
ไม่มี action อะไรเกิดขึ้นเลย ไม่ขึ้น success ไม่ขึ้น error
ข้อมูลที่กรอกคือชื่อ Asset Group ใหม่
ควรที่จะบันทึกเป็นข้อมูลปัจจุบันเลย
```

**กรณี: กดบันทึกหน้าแก้ไขสภาพถนนแล้วไม่เกิดอะไร**
```
/debug เมื่อเลือกเมนู Road > Condition แล้วแก้ไขข้อมูลสภาพถนน
กดปุ่มบันทึกแล้วไม่มี action อะไรเลย
ข้อมูลที่แก้คือ [ระบุ field ที่แก้]
ควรบันทึกข้อมูลที่แก้ไขเป็นข้อมูลปัจจุบันทันที
```

## 2. บันทึกข้อมูลแล้วไม่เกิดผล

### สถานการณ์
กดบันทึกแล้วขึ้น "สำเร็จ" แต่ข้อมูลไม่เปลี่ยน หรือกลับมาดูอีกทีข้อมูลเดิมๆ

### วิธีบอก Windsurf
```
/debug กดบันทึกที่หน้า [ชื่อหน้า] แล้วขึ้นว่าสำเร็จ
แต่ข้อมูลไม่ถูกบันทึกจริง กลับมาดูอีกทีก็ยังเป็นข้อมูลเดิม
ข้อมูลที่พยายามบันทึกคือ [ระบุข้อมูล]
```

### ตัวอย่างจริง

```
/debug กดบันทึกที่หน้า Setting > Department แล้วขึ้นว่าสำเร็จ
แต่กลับมาดูรายการอีกที ข้อมูลที่เพิ่มไม่ปรากฏ
พยายามเพิ่มแผนกใหม่ชื่อ "ฝ่ายบำรุงรักษา"
```

## 3. ข้อมูลแสดงผิดหรือไม่ตรง

### สถานการณ์
ข้อมูลที่แสดงบนหน้าจอไม่ตรงกับที่คาดไว้ เลขผิด สีผิด ค่าไม่ถูกต้อง

### วิธีบอก Windsurf
```
/debug ที่หน้า [ชื่อหน้า] ข้อมูลแสดงผิด
[ระบุว่าผิดยังไง] เช่น ค่า condition แสดงเป็นสีเขียวแต่ควรเป็นสีแดง
ข้อมูลที่คาดไว้คือ [ระบุค่าที่ถูกต้อง]
ข้อมูลที่แสดงคือ [ระบุค่าที่ผิด]
```

### ตัวอย่างจริง

```
/debug ที่หน้า Dashboard > Condition ข้อมูลสภาพถนนแสดงผิด
กราฟแสดงค่าเปอร์เซ็นต์รวมเกิน 100% ทั้งที่แต่ละช่วงไม่ควรรวมเกิน
คาดว่ารวมแล้วต้องได้ 100% พอดี
```

---

## 4. หน้าจอเปิดไม่ขึ้นหรือขึ้นขาว

### สถานการณ์
คลิกเมนูแล้วหน้าจอขาว หรือขึ้น error แล้วไม่โหลด

### วิธีบอก Windsurf
```
/debug เปิดเมนู [ชื่อเมนู] แล้วหน้าจอขาว ไม่มีข้อมูลแสดง
[ถ้ามี error บนจอ ให้บอกด้วย]
[ถ้ามีข้อมูลในช่อง URL ให้บอกด้วย]
```

### ตัวอย่างจริง

```
/debug เปิดเมนู Analysis > Strategic Summary แล้วหน้าจอขาว
ไม่มี error แสดงบนจอ แต่ไม่มีข้อมูลอะไรโหลดเลย
URL แสดงเป็น /analysis/strategic
```

## 5. อยากเพิ่มฟีเจอร์ใหม่

### สถานการณ์
อยากให้ระบบมีฟังก์ชันที่ยังไม่มี เช่น อยากค้นหาข้อมูลได้, อยากกรองข้อมูลได้, อยากส่งออกรายงานได้

### วิธีบอก Windsurf
```
/analysis อยากรู้ว่าตอนนี้หน้า [ชื่อหน้า] ทำอะไรได้บ้าง
แล้วต่อด้วย
/plan อยากเพิ่ม [ฟีเจอร์ที่ต้องการ] ที่หน้า [ชื่อหน้า]
```

### ตัวอย่างจริง

**อยากเพิ่มการค้นหาที่หน้ารายการถนน:**
```
/analysis อยากรู้ว่าหน้า Road List ตอนนี้ทำอะไรได้บ้าง
มีการค้นหาหรือยัง มีการกรองหรือยัง
```
```
/plan อยากเพิ่มช่องค้นหาที่หน้า Road List
ให้พิมพ์ชื่อถนนแล้วกรองผลลัพธ์ได้ทันที
```

**อยากเพิ่มปุ่มส่งออก Excel:**
```
/plan อยากเพิ่มปุ่ม "ส่งออก Excel" ที่หน้า Dashboard > Condition
ให้กดแล้วดาวน์โหลดข้อมูลสภาพถนนเป็นไฟล์ Excel ได้
```

## 6. อยากแก้ไขหน้าจอ/ปุ่ม/เมนู

### สถานการณ์
อยากเปลี่ยนข้อความ, ย้ายปุ่ม, เพิ่มตัวเลือกใน dropdown, เปลี่ยนสี, ซ่อน/แสดง field

### วิธีบอก Windsurf
```
/code ที่หน้า [ชื่อหน้า] อยาก [สิ่งที่อยากเปลี่ยน]
```

### ตัวอย่างจริง

**เปลี่ยนข้อความปุ่ม:**
```
/code เปลี่ยนข้อความปุ่ม "Submit" เป็น "บันทึก" ที่หน้า Setting > Sign
```

**เพิ่มตัวเลือกใน dropdown:**
```
/code เพิ่มตัวเลือก "ทางหลวงชนบท" ใน dropdown ประเภทถนน
ที่หน้า Road List
```

**ซ่อน field ที่ไม่ต้องการ:**
```
/code ซ่อนช่อง "Reference Code" ที่หน้า Setting > Department
เพราะไม่ได้ใช้งาน
```

**เปลี่ยนสีกราฟ:**
```
/code เปลี่ยนสีกราฟสภาพถนนจากสีแดง-เหลือง-เขียว
เป็นสีแดง-ส้ม-เหลือง-เขียว-น้ำเงิน ตามระดับ A-E
ที่หน้า Dashboard > Condition
```

## 7. ระบบช้าหรือค้าง

### สถานการณ์
โหลดนาน, กดแล้วรอนาน, หน้าค้าง, ข้อมูลโหลดไม่จบ

### วิธีบอก Windsurf
```
/debug ระบบช้ามากที่หน้า [ชื่อหน้า]
[ระบุว่าช้าตอนไหน] เช่น ตอนโหลดครั้งแรก / ตอนกดค้นหา / ตอนเปิดแผนที่
ข้อมูลประมาณ [จำนวน] รายการ
```

### ตัวอย่างจริง

```
/debug ระบบช้ามากที่หน้า Dashboard > Condition
ตอนเลือกปีแล้วกดค้นหา รอนานมากกว่าจะแสดงผล
ข้อมูลประมาณ 500 กม. ของถนนหลายสาย
```

## 8. อัปโหลดไฟล์ไม่ได้

### สถานการณ์
อัปโหลดรูป, ไฟล์ CSV, ไฟล์ Excel ไม่ผ่าน หรืออัปโหลดแล้วไฟล์หาย

### วิธีบอก Windsurf
```
/debug อัปโหลดไฟล์ที่หน้า [ชื่อหน้า] ไม่ได้
[ระบุประเภทไฟล์] เช่น รูป JPG, ไฟล์ CSV
[ระบุขนาดไฟล์ถ้ารู้]
[ระบุ error ที่ขึ้นถ้ามี]
```

### ตัวอย่างจริง

```
/debug อัปโหลดรูปป้ายจราจรที่หน้า Setting > Sign ไม่ได้
ไฟล์เป็น JPG ขนาด 5MB
ขึ้น error ว่า "File too large" แต่ไฟล์ไม่ได้ใหญ่มาก
```

## 9. รายงาน/ดาวน์โหลดไม่ได้

### สถานการณ์
กดปุ่มดาวน์โหลดรายงานแล้วไม่มีไฟล์, ไฟล์เปิดไม่ได้, ข้อมูลในรายงานผิด

### วิธีบอก Windsurf
```
/debug กดดาวน์โหลดรายงานที่หน้า [ชื่อหน้า] แล้ว [ปัญหา]
เช่น ไม่มีไฟล์ดาวน์โหลด / ไฟล์เปิดไม่ได้ / ข้อมูลในไฟล์ผิด
```

### ตัวอย่างจริง

```
/debug กดดาวน์โหลดรายงานสภาพถนนที่หน้า Analysis > Annual
แล้วไฟล์ Excel ที่ได้มาเปิดไม่ได้ ขึ้นว่าไฟล์เสีย
```

## 10. แจ้งเตือน/ข้อความผิดพลาด

### สถานการณ์
ขึ้นข้อความ error แดงๆ, ขึ้นข้อความที่อ่านไม่รู้เรื่อง, ขึ้นข้อความที่ไม่น่าจะถูก

### วิธีบอก Windsurf
```
/debug ที่หน้า [ชื่อหน้า] ขึ้นข้อความ error ว่า "[ข้อความ error ที่เห็น]"
เกิดขึ้นตอน [สิ่งที่ทำอยู่ตอนนั้น]
```

### ตัวอย่างจริง

```
/debug ที่หน้า Road > Asset ขึ้น error ว่า "Cannot read properties of undefined (reading 'id')"
เกิดขึ้นตอนคลิกดูรายละเอียดทรัพย์สินบนแผนที่
```

## 11. อยากให้ระบบทำอะไรใหม่ที่ยังไม่มี

### สถานการณ์
อยากให้มีฟีเจอร์ใหม่ที่ยังไม่มีในระบบเลย

### วิธีบอก Windsurf
```
/supreme อยากให้ระบบ [อธิบายสิ่งที่อยากได้]
เพราะ [เหตุผลที่ต้องการ]
ตอนนี้ต้อง [วิธีที่ทำอยู่ตอนนี้ถ้ามี]
```

### ตัวอย่างจริง

**อยากให้มีระบบแจ้งเตือน:**
```
/supreme อยากให้มีระบบแจ้งเตือนเมื่อสภาพถนนเปลี่ยนจากดีเป็นแย่
เพราะตอนนี้ต้องเข้าไปดู Dashboard เองทุกวัน
อยากให้ระบบส่ง email แจ้งเตือนอัตโนมัติได้
```

**อยากให้มีประวัติการแก้ไข:**
```
/supreme อยากให้มีประวัติการแก้ไขข้อมูล
เพราะตอนนี้ใครแก้อะไรไม่รู้ อยากรู้ว่าใครแก้ เมื่อไหร่ แก้อะไร
ควรมีทุกหน้าที่แก้ไขข้อมูลได้
```

## เคล็ดลับการบอกปัญหาให้ Windsurf เข้าใจง่าย

### ✅ บอกแบบนี้ (ดี)
- **บอกที่อยู่หน้าจอ**: "ที่หน้า Setting > Department"
- **บอกสิ่งที่ทำ**: "กดปุ่มบันทึก"
- **บอกสิ่งที่เกิดขึ้น**: "ไม่มีอะไรเกิดขึ้นเลย"
- **บอกสิ่งที่คาดไว้**: "ควรบันทึกข้อมูลแล้วขึ้นว่าสำเร็จ"
- **บอกข้อมูลที่ใส่**: "กรอกชื่อแผนกว่า ฝ่ายบำรุงรักษา"

### ❌ บอกแบบนี้ (ไม่ดี)
- "มันไม่ทำงาน" → ไม่รู้ว่าหน้าไหน ปุ่มไหน ทำอะไรอยู่
- "เจอบั๊ก" → ไม่รู้ว่าบั๊กยังไง
- "ช่วยแก้หน่อย" → ไม่รู้ว่าแก้อะไร

### สูตรมาตรฐาน
```
/[คำสั่ง] ที่หน้า [เมนู > หน้าย่อย] กด/ทำ [สิ่งที่ทำ] แล้ว [สิ่งที่เกิดขึ้น]
คาดว่าควร [สิ่งที่คาดไว้]
```

**ตัวอย่าง:**
```
/debug ที่หน้า Setting > Sign กดอัปโหลดรูปแล้วขึ้น error "File too large"
คาดว่าควรอัปโหลดรูป JPG ขนาด 5MB ได้
```

## คำสั่งทั้งหมดสรุป

| คำสั่ง | ใช้ทำอะไร | พูดง่ายๆ ว่า |
|--------|-----------|-------------|
| `/debug` | หาสาเหตุและแก้ปัญหา | "มีปัญหา ช่วยหาสาเหตุและแก้ที" |
| `/code` | เปลี่ยนแปลงหน้าจอ/ปุ่ม/ข้อความ | "ช่วยเปลี่ยน/เพิ่ม/ลบ ให้หน่อย" |
| `/plan` | วางแผนก่อนทำของใหญ่ | "อยากทำ [อะไร] ช่วยวางแผนก่อน" |
| `/analysis` | ดูว่าระบบตอนนี้เป็นยังไง | "ช่วยดูให้ว่าตอนนี้ทำอะไรได้บ้าง" |
| `/test` | ทดสอบว่าแก้แล้วใช้ได้ไหม | "ช่วยทดสอบว่าแก้แล้วใช้ได้ไหม" |
| `/deploy` | นำขึ้นระบบจริง | "ช่วย deploy ขึ้นระบบให้" |
| `/supreme` | ทำทุกอย่างครบจบ | "อยากได้ [ฟีเจอร์ใหม่] ทำให้ครบจบเลย" |
