# /continue: ทำงานต่อจากจุดที่หยุด

ใช้เมื่อ agent หยุดกลางงานเพราะ credit limit หมด หลังเพิ่ม credit แล้วให้พิมพ์ `/continue` เพื่อทำงานต่อ

## Step 1: อ่านสถานะล่าสุด

อ่านไฟล์เหล่านี้เพื่อรู้ว่าทำอะไรอยู่ ทำถึงไหนแล้ว:

1. **อ่าน `progress.txt`** (ถ้ามี) — ไฟล์สรุปความคืบหน้าที่ agent เขียนไว้
2. **อ่าน todo list** — เช็คว่ามี todo ค้างอยู่ไหม (status: `in_progress` หรือ `pending`)
3. **อ่านไฟล์ที่แก้ล่าสุด** — `git diff --stat` ดูว่าแก้ไฟล์ไหนไปแล้วบ้าง

```
// turbo
1. cat progress.txt (ถ้ามี)
2. ตรวจ todo list ปัจจุบัน
3. git diff --stat
4. git status
```

## Step 2: ระบุจุดที่หยุด

จากข้อมูล Step 1 ระบุ:

- **งานที่กำลังทำ**: [อะไร]
- **เสร็จแล้ว**: [อะไรบ้าง]
- **ค้างอยู่**: [อะไรที่ยังไม่เสร็จ]
- **ไฟล์ที่กำลังแก้อยู่**: [ไฟล์ไหน]

## Step 3: ทำงานต่อ

ทำงานจากจุดที่ค้าง **โดยไม่ต้องเริ่มใหม่**:

- ถ้ากำลังแก้ไฟล์อยู่ → อ่านไฟล์นั้น แล้วแก้ต่อ
- ถ้ากำลังรันคำสั่ง → รันต่อ
- ถ้ากำลังอยู่ใน workflow → ทำต่อจาก step ที่ค้าง
- ถ้ามี todo ค้าง → ทำต่อจาก todo ถัดไป

## Step 4: อัปเดตสถานะ

หลังทำงานต่อเสร็จแต่ละส่วน:
- อัปเดต todo list (mark completed)
- ถ้างานยังเหลือเยอะ → เขียน `progress.txt` สรุปสถานะปัจจุบันไว้ (กรณี credit หมดอีก)

---

## กฎสำคัญ

1. **ห้ามเริ่มใหม่**: อ่านสถานะก่อน แล้วทำต่อจากจุดที่หยุด
2. **ห้ามทำซ้ำ**: ถ้าอ่าน git diff แล้วเห็นว่าแก้ไขแล้ว → ข้าม ไม่ต้องแก้อีก
3. **เขียน progress.txt**: ถ้างานยังไม่เสร็จและอาจยาว → สรุปไว้เผื่อ continue รอบหน้า
4. **กระชับ**: อย่าอธิบายซ้ำ ทำต่อเลย
